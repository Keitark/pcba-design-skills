# ソースアダプターと検証レベル

[English](adapters.md) · [README に戻る](../README.ja.md)

Schematic Humanizer は共通のレビュー方法を、形式ごとのアダプターを通して適用します。ここでいうアダプターは必ずしもソフトウェアプラグインではなく、正本確認、安全な編集、レンダリング、検証方法の組み合わせです。

## 共通アダプテーション層

作業前に、すべてのアダプターが次の 4 点へ回答する必要があります。

1. **何が正本か:** ネイティブ設計ファイル、ジェネレーター、データベース、ネットリスト、レンダー済み資料のみのどれか。
2. **何を安全に編集できるか:** ネイティブソース、API、生成テーブル、交換形式、または編集不可のどれか。
3. **何を再現可能にレンダーできるか:** CLI、ヘッドレスエクスポート、アプリ自動操作、手動出力のどれか。
4. **何を証明できるか:** 厳密な接続、部分的な構造一致、見た目の一致のみのどれか。

入力が対応する最も強い証拠を使います。目視ベースの結果を、表現だけで「厳密」に格上げしてはいけません。

## 検証レベル

| レベル | 意味 | 最低限の報告 |
|---|---|---|
| A — 厳密 | 正本から変更前後の接続をエクスポートして比較可能 | ネット／部品数、変更ネットまたはピン所属、ERC／コンパイル結果、全ページ目視確認 |
| B — 構造化 | 編集可能な構造化ソースはあるが、厳密なネイティブ比較は不可 | パーサー／エクスポート結果、安定 ID、警告、全ページ目視確認、残る不確実性 |
| C — 目視ベース | PDF／画像だけが信頼できる入力 | ページ一覧、追跡した機能経路、判読不能／曖昧箇所。接続維持は主張しない |

## KiCad

**通常レベル: A。第一級対応。**

- `.kicad_sch` をジェネレーターが所有する場合はジェネレーターを優先し、それ以外は対応ツールでネイティブソースを編集します。
- 編集前にネットリストと ERC を保存します。
- KiCad の意味に従って実配線、バスエントリー、階層シート、ラベルを使用します。
- 利用可能ならインストール済み KiCad CLI で再生成・エクスポートします。
- 部品番号、ピンとネットの所属、ネット名、ERC、レンダー画像を比較します。
- リポジトリの指示が別の正本を指定している場合、生成済み `.kicad_sch` を直接編集しません。

## Altium Designer

**通常レベル: A または B。**

- ネイティブ API／自動化、または明示的に対応した ASCII／交換形式を優先します。
- コピーを保存し、変更前後のネットリストまたはプロジェクトコンパイル結果を出力します。
- 部品固有 ID とシート関係を維持します。
- PDF と BOM しかない場合は、ネイティブ接続維持を主張せずレベル C へ下げます。

## EasyEDA

**通常レベル: エディションとエクスポート権限により A または B。**

- Standard／Pro の別と、プロジェクト JSON／ネイティブソースの利用可否を確認します。
- 可能ならネイティブのネットリストと PDF/SVG を証拠に使います。
- 見た目で配線端が接していても接続済みとは限りません。エクスポートでジャンクションの意味を確認します。
- 変換・ダウンロードしたプロジェクトは新しいソースとして扱い、変換制約を記録します。

## Autodesk EAGLE / Fusion Electronics

**通常レベル: A または B。**

- EAGLE XML `.sch` は構造化検証に利用できます。Fusion 管理設計はアプリからのエクスポートが必要な場合があります。
- deviceset、gate、pin swap、電源シンボルの意味を維持します。
- 変更前後にネットリスト出力またはネイティブ ERC を実行します。
- 見た目だけを理由にライブラリ ID を書き換えません。

## gEDA / Lepton EDA

**通常レベル: A または B。**

- テキスト回路図は解析できますが、ネットの意味はインストール済みツールとシンボルライブラリに依存します。
- 利用可能なら `gnetlist`／Lepton 相当ツールとネイティブ検査を使います。
- refdes、slot、属性、ネット名を維持します。
- 一致するライブラリ環境でレンダーします。シンボル不足時は目視検証が不完全です。

## SPICE／単体ネットリスト

**通常レベル: 記載接続は A、回路図の意図は B。**

- ノードと端子についてネットリストを正本として扱います。
- 記載されていない接続を作らず、機能図を構成します。
- サブサーキット展開、グローバルノード、エイリアス、モデルのピン対応を記録します。
- 再構成した接続を元のネットリストと比較します。
- 電源の意図、コネクターの意味、隠しピン、機能グループは人が確認します。

## PDF／画像のみ

**レベル: C のみ。**

- 入手可能な最高解像度の元ページを使います。
- 見える配線、ジャンクション、ラベル、ページ参照、機能ブロックを追跡します。
- 切れている、ぼやけている、隠れている、曖昧な接続を明示します。
- 注釈または再作図計画を作成し、「正本」となる編集可能回路図を無断で作りません。
- 接続維持を独立検証できないことを明記します。

## アダプターを追加する

アダプターの追加では、正本の見つけ方、安全な編集手段、再現可能なレンダー手段、接続検査、想定される失敗、正直に主張できる最大検証レベルを記載してください。[CONTRIBUTING.md](../CONTRIBUTING.md) も参照してください。
